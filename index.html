O HTML "Não Semântico" (O jeito antigo)

Antigamente, quando a gente queria criar uma página web, usávamos elementos genéricos para tudo, principalmente a <div>.

  • Precisa de um cabeçalho? <div id="cabeçalho">.
  • Um menu de navegação? <div id="menu">
  • O conteúdo principal? <div id="conteudo">
  • Um rodapé? <div id="rodape">

Funciona? Sim. O site fica com a aparência que você quer (com a ajuda do CSS). Mas, para o navegador, para as ferramentas de busca (como o Google) e para leitores de tela (usados por pessoas com deficiência visual), essa estrutura é como um monte de caixas sem etiqueta. Ninguém sabe o que tem dentro de cada <div> sem ter que olhar o id ou a class. É como um documento de texto sem títulos ou parágrafos, só um monte de texto corrido.

O HTML Semântico (O jeito certo e moderno)

O HTML Semântico resolve isso. Ele introduziu novas tags que têm um significado, uma semântica. Elas dizem exatamente o que são.

Vamos refazer a mesma estrutura de antes, mas com tags semânticas:

  • Para o cabeçalho, em vez de <div id="cabeçalho">, usamos <header>.
  • Para o menu de navegação, em vez de <div id="menu">, usamos <nav>.
  • Para o conteúdo principal, usamos <main>.
  • Para seções importantes dentro do conteúdo, usamos <section>.
  • Para um conteúdo mais independente, como um post de blog, usamos <article>.
  • Para uma barra lateral, usamos <aside>.
  • E para o rodapé, usamos <footer>.

Tags Semânticas Principais

  1. <Header>

    • O que é? Representa o cabeçalho de uma página ou de uma seção. É o "cartão de visitas" do seu site.
    • O que colocar dentro? Geralmente contém o logo da empresa, o título principal da página (com <h1>), um campo de busca e, muitas vezes, a navegação principal (<nav>).
    • Ponto importante: Você pode ter mais de um <header> por página. Por exemplo, um <article> (como um post de blog) pode ter seu próprio cabeçalho com o título do post e o nome do autor.


  2. <Nav>

    • O que é? A tag <nav> (de "navigation") é usada para agrupar os links de navegação mais importantes do seu site.
    • O que colocar dentro? Menus principais, links para as seções mais importantes (como "Home", "Sobre", "Produtos", "Contato"), ou até mesmo uma paginação em um blog.
    • Ponto importante: Não use <nav> para qualquer grupo de links. Se você tem uma lista de links no rodapé para a política de privacidade e termos de uso, não precisa envolvê-los em uma <nav>. A tag é reservada para a navegação principal.


  3. <Main>

    • O que é? É o coração da sua página. Representa o conteúdo principal e único daquele documento.
    • O que colocar dentro? O conteúdo que o usuário veio ver: o texto de um artigo, as fotos de uma galeria, os produtos de uma loja, o formulário principal, etc.
    • Ponto importante: Só pode existir uma tag <main> por página. Ela não deve incluir conteúdos que se repetem em todo o site, como o cabeçalho, o rodapé ou a barra lateral.


  4. <Section>

    • O que é? Usada para agrupar conteúdos que têm um tema em comum, como um capítulo de um livro. É uma seção temática dentro do seu conteúdo.
    • O que colocar dentro? Por exemplo, em uma página inicial, você pode ter uma <section> para "Nossos Serviços", outra para "Depoimentos de Clientes" e uma terceira para "Últimas Notícias".
    • Ponto importante: Uma <section> quase sempre deve ter um título (como <h2>, <h3>, etc.) para identificar do que se trata aquele bloco de conteúdo. Se não faz sentido dar um título, talvez uma <div> seja mais apropriada.


  5. <Article>

    • O que é? Representa um conteúdo completo e independente, que faria sentido por si só se fosse distribuído em outro lugar (como em um feed de notícias ou RSS).
    • O que colocar dentro? Um post de blog, uma notícia de jornal, um comentário de um usuário em um fórum, um widget de previsão do tempo.
    • Ponto importante: A principal diferença entre <article> e <section> é a independência. Um <article> pode ser "recortado" e colado em outro site e ainda fazer sentido. Uma <section> é apenas uma parte da página atual. Você pode ter vários <article>s dentro de uma <section> (ex: uma seção de "Notícias" com vários artigos).


  6. <Aside>

    • O que é? Representa um conteúdo que está relacionado ao conteúdo principal, mas não é parte essencial dele. É uma informação "ao lado".
    • O que colocar dentro? A famosa "barra lateral" (sidebar) com links relacionados, biografias curtas, publicidade, ou uma caixa de "saiba mais" dentro de um artigo.
    • Ponto importante: A tag <aside> pode estar dentro do <main> (se for relacionada ao conteúdo principal) ou fora dele (se for relacionada ao site como um todo).


  7. <Footer>

    • O que é? Representa o rodapé de uma página ou de uma seção.
    • O que colocar dentro? Informações de copyright, links para documentos (termos de uso, política de privacidade), informações de contato, mapa do site, etc.
    • Ponto importante: Assim como o <header>, você pode ter mais de um <footer> por página. Um <article> pode ter seu próprio rodapé com informações sobre o autor ou tags relacionadas ao post.
